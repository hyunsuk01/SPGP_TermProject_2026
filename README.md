# Memory Route

# 게임 컨셉

## High Concept
플레이어의 이동 입력을 기록하고, 이후 해당 입력을 순차적으로 수행하는 분신과 동시에 움직이며 퍼즐을 해결하는 2D 퍼즐 게임

---

## 핵심 메카닉

### 1. 입력 기록
- 플레이어는 제한된 횟수(N번) 동안 이동 입력을 수행
- 입력은 순서대로 저장됨

### 2. 분신 재생
- 플레이어가 이동할 때마다 저장된 입력을 순차적으로 수행
- 과거의 행동을 따라오는 형태로 동작

### 3. 동시 진행
- 플레이어와 분신이 동시에 움직이며 퍼즐을 해결

### 4. 퍼즐 상호작용
- 버튼, 문, 발판 등을 활용하여 협력 퍼즐 해결

---

# 개발 범위

## 플레이어 시스템
- 이동 (상 / 하 / 좌 / 우)
- 입력 기록 (최대 5~10회)
- 입력 기록 이후에도 계속 이동 가능

## 분신 시스템
- 플레이어가 기록한 입력을 순차적으로 수행
- 플레이어의 현재 움직임에 따라 지연된 입력을 실행
- 최대 1개 생성

## 퍼즐 요소
- 버튼: 밟으면 문이 열림
- 문: 특정 조건에서 열림
- 벽: 이동 불가
- 사다리: 상하 이동 가능

## 실패 조건
- 분신과 충돌
- 벽 또는 이동 불가 지역으로 이동 시도
- 맵 밖으로 이동

## 성공 조건
- 플레이어가 목표 지점 도달

## 스테이지 구성
- 여러 스테이지로 구성
- 진행될수록 퍼즐 난이도가 점진적으로 증가

## 씬 구성
- 시작 화면
- 게임 화면
- 클리어 화면

---

# 예상 게임 실행 흐름

## 전체 흐름
1. 시작 화면 → 게임 시작  
2. 스테이지 진입  
3. 플레이어 이동 입력 기록 (최대 N회)  
4. 이후 플레이어는 계속 이동 가능  
5. 분신이 저장된 입력을 순차적으로 수행하며 따라옴  
6. 플레이어 + 분신 동시 이동  
7. 퍼즐 해결 (버튼 활성화 등)  
8. 목표 지점 도달 → 클리어  
9. 다음 스테이지 진행  

---

- 버튼을 밟으면 문으로 가는 길이 생김  
- 버튼을 밟지 않으면 길이 사라짐

해결 방법:
1. 플레이어가 버튼 위치까지 이동하며 입력 기록  
2. 이후 분신이 해당 위치로 이동하여 버튼 유지  
3. 플레이어는 다른 경로로 이동  
4. 문을 통과하여 클리어

# 개발 일정 (8주)

한 라운드 완성 → 확장 방식

## 1주차
- 프로젝트 초기 설정
- 기본 이동 시스템 구현 (상하좌우)

## 2주차
- 입력 기록 시스템 구현 (배열/리스트 저장)
- 입력 횟수 제한 기능 추가

## 3주차
- 분신(클론) 시스템 구현
- 입력 재생 기능 구현

## 4주차
- 플레이어 + 분신 동시 이동 처리
- 충돌 처리 (벽, 맵 경계)

## 5주차
- 퍼즐 오브젝트 구현 (버튼, 문)
- 1개 완성된 라운드 제작  

-> 완전 플레이 가능한 상태 목표

## 6주차
- 추가 스테이지 제작
- 퍼즐 난이도 확장

## 7주차
- 추가 스테이지 제작
- UI (시작/클리어 화면) 구현

## 8주차
- 버그 수정 및 밸런싱
- 전체 흐름 완성

---

# 차별점

- 단순 이동이 아닌 입력 기반 퍼즐 설계
- 플레이어 행동 자체가 퍼즐 요소가 됨
- 분신과의 협력을 통한 전략적 플레이

---

# 구현 제한 (설계 의도)

- 분신은 1개만 생성 가능
- 입력 횟수 제한 (최대 10회)
